# Prompts IA para Personajes y Sprites

Fecha: 2026-03-27

## Uso recomendado

Primero usar los prompts de `concept`.
Solo cuando el concept cierre, pasar a los prompts de `sprite`.

## Template base de concept

```text
Use case: stylized-concept
Asset type: game character concept art
Primary request: full body character concept for a 2D beat'em up set inside the Buenos Aires San Martin commuter train
Scene/backdrop: neutral or transparent background, no environment details
Subject: <PERSONAJE>
Style/medium: semi-realistic digital illustration, grounded urban realism, readable silhouette, not anime, not cartoon, not photoreal
Composition/framing: full body, centered, feet visible, enough top margin, slight 3/4 view
Lighting/mood: warm commuter train interior lighting, soft top light, subtle shadow under the character
Color palette: worn blues, browns, metal grays, dirty creams, restrained orange accents
Materials/textures: slightly worn clothes, practical fabrics, believable folds, daily-use texture
Constraints: transparent or clean neutral background, no text, no watermark, no cut limbs, no props outside character identity
Avoid: anime proportions, exaggerated caricature, sci-fi elements, fantasy elements, glossy studio fashion look
```

## Template base de sprite frame

```text
Use case: stylized-concept
Asset type: 2D game sprite frame
Primary request: single full body sprite frame for a side-scrolling beat'em up character
Scene/backdrop: transparent background
Subject: <PERSONAJE> in pose <POSE>
Style/medium: semi-realistic digital illustration for game sprite production, consistent with previous character sheet
Composition/framing: full body, centered, same scale and camera as the approved concept, feet fully visible, clean silhouette
Lighting/mood: warm train interior lighting, consistent from frame to frame
Color palette: keep approved palette
Materials/textures: same clothing and accessories as approved concept
Constraints: transparent background, no text, no watermark, preserve outfit and proportions exactly, readable at small size
Avoid: dramatic camera changes, extra accessories, cropped body parts, background objects
```

## Prompts de sprite frame

Usar el mismo prompt base y reemplazar:

- `<PERSONAJE>`
- `<POSE>`

Poses recomendadas:

- `idle-01`: neutral standing, subtle tension
- `idle-02`: same stance with slight weight shift
- `walk-01`: left leg forward, active torso
- `walk-02`: right leg forward, active torso
- `attack-01`: clear readable attack silhouette
- `hurt-01`: torso recoils, defensive pain reaction
- `defeated-01`: collapsed or staggered defeat pose, still readable

## Prompt de consistency pass

Cuando ya tengas un concept aprobado, usar este prompt para los frames:

```text
Use case: stylized-concept
Asset type: consistent sprite production
Primary request: create one full body sprite frame for the already approved character design, preserving face, outfit, proportions, palette and silhouette exactly
Scene/backdrop: transparent background
Subject: <PERSONAJE> in pose <POSE>
Style/medium: semi-realistic digital illustration for beat'em up game sprite, matching the approved base concept exactly
Composition/framing: full body, centered, same camera, same scale, feet visible, clean silhouette
Lighting/mood: same warm train interior light as the approved concept
Constraints: preserve design exactly, no new accessories, no new clothing details, no text, no watermark, transparent background
Avoid: style drift, anatomy drift, palette drift, camera drift, cropped limbs
```
